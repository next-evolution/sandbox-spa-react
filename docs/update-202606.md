## 各アップデート詳細

### 1. TypeScript 5 → 6

**難易度: 中**

#### 主な破壊的変更

- `enum` / `namespace` など「消去できない構文（non-erasable syntax）」がデフォルトで警告対象になる
  - `const` オブジェクト + 型への置き換えが必要になる場合あり
- 型チェックの厳格化により既存コードで型エラーが増える可能性あり
- `isolatedModules` 相当の挙動がデフォルト化

#### 対応手順

```bash
npm install -D typescript@^6.0.3
npm run build  # 型エラーを洗い出す
```

型エラーを1件ずつ修正する。

---

### 2. Vite 5 → 8 ＋ @vitejs/plugin-react 4 → 6

**難易度: 低〜中**

#### 主な変更

- **Node.js 20 以上が必要**（v5 は Node.js 18+）
- `@vitejs/plugin-react` v6 は Vite 8 専用のため、必ずセットでアップデートする
- 現在残存している `esbuild` の脆弱性（moderate）も同時に解消される
- `vite.config.ts` の設定 API に一部変更がある可能性あり

#### 対応手順

```bash
npm install -D vite@^8.0.16 @vitejs/plugin-react@^6.0.2
npm run build
npm run dev  # 動作確認
```

`vite.config.ts` の差分は公式マイグレーションガイドで確認すること。

---

### 3. React 18 → 19 ＋ @types/react / @types/react-dom

**難易度: 中〜高**

#### 主な破壊的変更

| 変更点 | 内容 |
|---|---|
| `forwardRef` 廃止 | `ref` を通常の props として受け取る形式に変更 |
| `Context.Provider` 不要 | `<MyContext value={...}>` を直接使えるようになった |
| `useRef` の初期値 | 必須に変更（`useRef<Type>(null)` は引き続き OK） |
| `defaultProps`（関数コンポーネント） | 廃止 |
| `@types/react` / `@types/react-dom` | v19 へ合わせて更新が必要 |

#### 主な新機能（使えるようになるもの）

- `use()` フック（Promise / Context を同期的に扱える）
- `useOptimistic`（楽観的 UI 更新）
- `useActionState`（旧 `useFormState`）

#### 対応手順

```bash
npm install react@^19 react-dom@^19
npm install -D @types/react@^19 @types/react-dom@^19
npm run build  # 型エラー・deprecation を確認
```

`forwardRef` を使っているコンポーネントは要確認・修正。

---

### 4. react-router-dom 6 → 7

**難易度: 低〜中**

#### 前提

v7 は **「ライブラリモード」** と **「フレームワークモード（旧 Remix）」** の 2 系統がある。
本プロジェクトは SPA のため **ライブラリモード** を選択する。

#### 主な変更

- v6 の `future` フラグ（`v7_relativeSplatPath` / `v7_fetcherPersist` 等）を全有効化した状態が v7 のデフォルトになった
- `<Route>` の TypeScript 型推論が強化
- `createBrowserRouter` + `RouterProvider` が引き続き推奨

#### 対応手順

```bash
# ステップ1: v6 のまま future フラグを全て有効化して挙動確認
# ステップ2: 問題なければ v7 へアップデート
npm install react-router-dom@^7
npm run build
npm run dev  # ルーティング動作確認
```

v6 で `future` フラグなしの `<BrowserRouter>` を使っている場合は、フラグを追加してから v7 に移行するのが安全。

## 推奨実施順序

依存関係の観点から、以下の順で1ステップずつ進める。
各ステップで `npm run build` と動作確認を行い、問題がないことを確認してから次へ進むこと。

```
Step 1: TypeScript 6
         ↓
Step 2: Vite 8 + @vitejs/plugin-react 6
         ↓
Step 3: React 19 + @types/react 19 + @types/react-dom 19
         ↓
Step 4: react-router-dom 7
```

### 理由

- TypeScript を先にアップグレードすることで、後続ライブラリのアップデート時に型エラーの原因を切り分けやすくなる
- Vite は React とは独立しているため、React より前に完了させておくと差し戻しが容易
- react-router-dom は React 本体が安定してから変更するのが安全
